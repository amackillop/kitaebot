# NixOS VM

## Purpose

The agent runs inside a NixOS virtual machine. This provides isolation, reproducibility, and a consistent deployment target whether running locally or in the cloud.

## Why a VM?

1. **Isolation** — Agent can't affect host system
2. **Reproducibility** — Same environment everywhere
3. **Security** — VM is the security boundary, not regex guards
4. **Simplicity** — One artifact to deploy, backup, restore

## Directory Structure

```
/nix/
├── flake.nix              # Entry point, defines outputs
├── configuration.nix      # Base system configuration
├── kitaebot.nix           # Agent service module
└── workspace/             # Template files for workspace
    ├── SOUL.md
    ├── USER.md
    ├── AGENTS.md
    └── ...
```

## Flake Structure

```nix
# flake.nix
{
  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixos-24.05";
    crane.url = "github:ipetkov/crane";
    fenix.url = "github:nix-community/fenix";
  };

  outputs = { self, nixpkgs, crane, fenix, ... }: {
    # The VM configuration
    nixosConfigurations.kitaebot = nixpkgs.lib.nixosSystem {
      system = "x86_64-linux";
      modules = [
        ./configuration.nix
        ./kitaebot.nix
      ];
    };

    # Build just the package
    packages.x86_64-linux.default = <kitaebot derivation>;

    # Development shell
    devShells.x86_64-linux.default = <dev shell>;
  };
}
```

## Base Configuration

```nix
# configuration.nix
{ config, pkgs, ... }:

{
  # Minimal system
  system.stateVersion = "25.11";

  # Boot configuration for VM
  boot.loader.grub.enable = false;
  boot.loader.generic-extlinux-compatible.enable = true;

  # Network (host-only or NAT)
  networking.hostName = "kitaebot";
  networking.firewall.enable = true;
  networking.firewall.allowedTCPPorts = [ 22 ];

  # SSH access
  services.openssh = {
    enable = true;
    settings.PasswordAuthentication = false;
  };

  # User account
  users.users.kitaebot = {
    isNormalUser = true;
    home = "/home/kitaebot";
    extraGroups = [ "wheel" ];
    openssh.authorizedKeys.keys = [
      # User's SSH public key goes here
    ];
  };

  # Basic packages
  environment.systemPackages = with pkgs; [
    vim
    git
    curl
    htop
  ];
}
```

## Agent Service Module

```nix
# kitaebot.nix
{ config, pkgs, lib, ... }:

let
  kitaebot = <package derivation>;
  workspaceTemplates = ./workspace;
in {
  # Create workspace directory
  systemd.tmpfiles.rules = [
    "d /var/lib/kitaebot 0755 kitaebot kitaebot -"
  ];

  # Copy templates on first boot
  system.activationScripts.kitaebot-workspace = ''
    if [ ! -f /var/lib/kitaebot/SOUL.md ]; then
      cp -r ${workspaceTemplates}/* /var/lib/kitaebot/
      chown -R kitaebot:kitaebot /var/lib/kitaebot
    fi
  '';

  # Agent service
  systemd.services.kitaebot = {
    description = "Kitaebot agent daemon";
    wantedBy = [ "multi-user.target" ];
    after = [ "network.target" ];

    serviceConfig = {
      Type = "simple";
      User = "kitaebot";
      WorkingDirectory = "/var/lib/kitaebot";
      ExecStart = "${kitaebot}/bin/kitaebot daemon";
      Restart = "always";
      RestartSec = "5s";

      # Security hardening
      NoNewPrivileges = true;
      ProtectSystem = "strict";
      ProtectHome = true;
      ReadWritePaths = [ "/var/lib/kitaebot" ];
    };

    environment = {
      OPENROUTER_API_KEY = ""; # Set via EnvironmentFile
    };
  };

  # Heartbeat timer
  systemd.timers.kitaebot-heartbeat = {
    wantedBy = [ "timers.target" ];
    timerConfig = {
      OnBootSec = "5min";
      OnUnitActiveSec = "30min";
      Persistent = true;
    };
  };

  systemd.services.kitaebot-heartbeat = {
    description = "Kitaebot heartbeat";
    serviceConfig = {
      Type = "oneshot";
      User = "kitaebot";
      WorkingDirectory = "/var/lib/kitaebot";
      ExecStart = "${kitaebot}/bin/kitaebot heartbeat";
    };
  };
}
```

## Building the VM

```bash
# Build VM image
nix build .#nixosConfigurations.kitaebot.config.system.build.vm

# Result is a script that launches QEMU
./result/bin/run-kitaebot-vm
```

## Running Locally

```bash
# Start VM (QEMU)
./result/bin/run-kitaebot-vm

# SSH in (default port forwarding)
ssh -p 2222 kitaebot@localhost

# Run the agent
kitaebot
```

## Secrets Management

API keys should not be in the Nix configuration. Options:

**Option 1: Environment file**
```nix
systemd.services.kitaebot.serviceConfig.EnvironmentFile = "/var/lib/kitaebot/secrets.env";
```

```bash
# /var/lib/kitaebot/secrets.env
OPENROUTER_API_KEY=sk-or-...
```

**Option 2: sops-nix** (more secure)
```nix
sops.secrets.openrouter-api-key = {
  owner = "kitaebot";
};

systemd.services.kitaebot.serviceConfig.EnvironmentFile =
  config.sops.secrets.openrouter-api-key.path;
```

## Cloud Deployment

Same configuration works for cloud:

```bash
# Deploy to server using nixos-anywhere
nixos-anywhere --flake .#kitaebot root@server.example.com
```

## Resource Limits

```nix
systemd.services.kitaebot.serviceConfig = {
  MemoryMax = "1G";
  CPUQuota = "50%";
};
```

## VM Characteristics

| Property | Value | Rationale |
|----------|-------|-----------|
| RAM | 512MB-1GB | Minimal, agent is lightweight |
| Disk | 2GB+ | Workspace + packages |
| CPUs | 1-2 | Not CPU intensive |
| Network | NAT or host-only | Outbound for API, SSH inbound |
